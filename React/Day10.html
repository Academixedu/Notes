Examples include `number`, `string`, `boolean`, `null`, `undefined`, `symbol`, and `bigint`. ```javascript // Using const with value types const number = 42; const string = 'Hello, World!'; const boolean = true; // Trying to reassign the variable will cause an error // number = 50; // Error: Assignment to constant variable. ``` ### Reference Types Reference types store a reference (or address) to the actual data in memory. Examples include `objects`, `arrays`, and `functions`. #### Example with Objects ```javascript // Using const with reference types const person = { name: 'Alice', age: 30 }; // The reference 'person' cannot be changed // person = { name: 'Bob', age: 25 }; // Error: Assignment to constant variable. // However, the properties of the object can be changed person.name = 'Bob'; // This is allowed person.age = 25; // This is allowed console.log(person); // { name: 'Bob', age: 25 } ``` #### Example with Arrays ```javascript // Using const with arrays const numbers = [1, 2, 3, 4, 5]; // The reference 'numbers' cannot be changed // numbers = [6, 7, 8, 9, 10]; // Error: Assignment to constant variable. // However, the elements of the array can be changed numbers.push(6); // This is allowed numbers[0] = 0; // This is allowed console.log(numbers); // [0, 2, 3, 4, 5, 6] ``` ### Explanation - **Value Types with `const`**: When you declare a variable with `const` and assign it a value type (like a number or string), the variable holds the actual value. Since `const` prevents reassignment, you cannot change the value stored in the variable. - **Reference Types with `const`**: When you declare a variable with `const` and assign it a reference type (like an object or array), the variable holds a reference to the location in memory where the data is stored. `const` prevents you from changing the reference itself, but it does not prevent you from modifying the data at that location. ### Full Example Program Here's a complete example that demonstrates both concepts: ```javascript // Value types const myNumber = 10; const myString = 'Hello'; const myBoolean = true; console.log('Value Types Before Change:'); console.log(myNumber); // 10 console.log(myString); // Hello console.log(myBoolean); // true // Uncommenting the following lines will cause an error because 'const' prevents reassignment // myNumber = 20; // myString = 'World'; // myBoolean = false; // Reference types const myObject = { key1: 'value1', key2: 'value2' }; const myArray = [1, 2, 3, 4, 5]; console.log('Reference Types Before Change:'); console.log(myObject); // { key1: 'value1', key2: 'value2' } console.log(myArray); // [1, 2, 3, 4, 5] // Modifying properties of the object myObject.key1 = 'newValue1'; myObject.key3 = 'value3'; // Modifying elements of the array myArray.push(6); myArray[0] = 0; console.log('Reference Types After Change:'); console.log(myObject); // { key1: 'newValue1', key2: 'value2', key3: 'value3' } console.log(myArray); // [0, 2, 3, 4, 5, 6] // Uncommenting the following lines will cause an error because 'const' prevents reassignment of the reference // myObject = { newKey: 'newValue' }; // myArray = [7, 8, 9]; ``` ### Summary - **Value Types**: Using `const` with value types (e.g., numbers, strings) means the variable cannot be reassigned to a new value. - **Reference Types**: Using `const` with reference types (e.g., objects, arrays) means the reference cannot be changed, but the properties or elements of the referenced data can be modified. #### Prompt Engineering Practice Generate a JavaScript module using Axios to fetch movie data from TheMovieDatabase (TMDB) API.
